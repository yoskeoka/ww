Place active execution plans here.

Use `<sequence>-<name>.md` filenames, where the suffix still matches the branch
description after the `<type>/` prefix. For example, branch
`plan/remove-docker-from-integration-tests` maps to
`docs/exec-plan/todo/0031-remove-docker-from-integration-tests.md`.

Use the next available sequence number so active plan creation order stays visible
without checking Git history.

When a plan is completed and the PR is ready, move the file to `../done/`.
